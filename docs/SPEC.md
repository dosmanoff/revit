# Спецификация плагина «Wall Reinforcement Suite»

> Версия документа: 0.1 (draft)
> Целевая платформа: Autodesk Revit **2025**, .NET 8, C#
> Референс по функциональности: [BeSmart Concrete Collection / Precast Concrete](https://docs.besmart.software/3d-modeling-and-design/precast-concrete/feature-descriptions/wall-reinforcement.md)

---

## 1. Контекст и цели

### 1.1 Назначение
Плагин автоматизирует трудоёмкие операции конструктора-проектировщика ЖБК в Revit:
1. Армирование монолитных и сборных железобетонных стен по конфигурации.
2. Создание выпусков (стартовых стержней, dowels) из нижележащих конструкций — фундаментных плит, стен нижнего яруса, перекрытий — в армируемую стену.
3. Назначение арматуре параметров (марка стены, ярус, этаж, позиция, секция, тип армирования, ...) для последующей фильтрации на видах и в спецификациях.
4. Создание видов и графических деталей для отображения армирования стен: поперечные сечения, продольные развёртки, фрагменты, узлы.
5. Создание спецификаций арматуры (ведомость деталей арматуры, форма С-49, ВРС, выборка из проекта).
6. Управление **конфигурациями армирования** через диалог редактора: формы стержней, диаметры, шаги, защитные слои, типы материалов, правила привязки параметров.

### 1.2 Целевая аудитория
- Инженер-конструктор ЖБК (основной).
- BIM-координатор (вспомогательный — настройка шаблонов).
- Технолог сборных конструкций (вторичный сценарий для precast).

### 1.3 Принципы продукта
- **Конфигурация важнее кода.** Все нормативы, диаметры, шаги, формы — в JSON-конфигах, которые можно редактировать и версионировать.
- **Идемпотентность.** Повторный запуск армирования на той же стене с той же конфигурацией не должен накапливать дубли арматуры.
- **Прозрачность результата.** Каждый созданный стержень несёт ссылку на правило (Rule ID) из конфига, по которому он создан.
- **Откатываемость.** Любая операция выполняется в одной транзакции и должна корректно откатываться через стандартный Undo Revit.

## 2. Глоссарий

| Термин | Описание |
|---|---|
| **Стена-приёмник** (host wall) | Стена, в которую размещается арматура. |
| **Стена-источник** (source) | Конструкция уровнем ниже, из которой выпускается стартовая арматура (фундамент, нижестоящая стена, перекрытие). |
| **Выпуск / Dowel / Starter bar** | Г- или прямой стержень, заходящий в конструкцию-источник на анкеровку и выходящий в стену-приёмник. |
| **Конфигурация армирования** (RebarConfig) | Именованный набор правил армирования стены (внешние/внутренние сетки, периметр, проёмы, углы, T-узлы, выпуски). |
| **Правило армирования** (RebarRule) | Атомарное правило внутри конфигурации, описывающее один тип стержней (например, «вертикальные внешней сетки Ø12 шаг 200»). |
| **Wall Link** | Привязка стены к конфигурации армирования — хранится в стене как параметр + в централизованном хранилище плагина. |
| **Партиция** (Partition) | Параметр Revit, используемый для группировки стержней при нумерации. |
| **Марка стены** | Пользовательский параметр-тип стены, определяющий «класс» армирования (например, «СТ-1», «СТ-2»). |

## 3. Функциональные модули

### M1. Армирование стен (Wall Reinforcement)
**Цель.** Разместить арматуру в одной или нескольких стенах согласно выбранной конфигурации.

**Входы.**
- Выбор стен в плане (single / multi-select).
- Имя конфигурации (или Wall Link, если уже привязан).
- Активный документ Revit.

**Выходы.**
- Набор объектов `Autodesk.Revit.DB.Structure.Rebar` и `RebarInSystem`, привязанных к стене.
- (Опционально) `FabricSheet` / `FabricArea` для сеточного армирования.
- Запись в Extensible Storage стены: `WallLinkSchema { ConfigId, ConfigVersion, RunTimestamp, RuleIds[] }`.

**Подмодули правил (взято из BeSmart как референс):**
- **External / Internal Reinforcement** — основные сетки (вертикальная + горизонтальная) от внешней / внутренней грани слоя стены.
- **Perimeter Reinforcement** — окантовка периметра: краевые продольные, диагональные в углах, U/O-хомуты по вертикальным и горизонтальным граням.
- **Opening / Door / Window Reinforcement** — окантовка проёмов (краевые, диагональные, U/O-хомуты).
- **L-Corner Reinforcement** — арматура в L-сопряжениях стен (внутренний/внешний угол, Bar1…Bar5).
- **T-Connection Reinforcement** — арматура в T-сопряжениях стен.
- **Additional Edge Rebar** — дополнительные стержни на выбранных вертикальных/горизонтальных гранях (прямые, L, U).
- **Additional Reinforcement** — произвольные пользовательские стержни на гранях (U, L, O, прямые).
- **Face Rebar** — стержни «по грани» как универсальное правило (Wall, Beam, Column).

**Поддерживаемые операции над уже армированной стеной:**
- **Update** — пересоздать весь набор арматуры по текущей конфигурации.
- **Partial Update** — обновить только выбранные правила (например, только сетки, не трогая периметр) — *MVP: вне scope, см. ROADMAP*.
- **Modify** — точечная корректировка отдельных стержней с сохранением связи с правилом.
- **Delete** — удалить всю арматуру, созданную плагином, в выбранных стенах.

**Граничные случаи.**
- Стена многослойная (sandwich) — целевой слой выбирается в конфигурации или диалоге.
- Стена непрямая (изогнутая, наклонная) — поддержка обязательная для v1.0, MVP — только прямые вертикальные.
- Стены с проёмами и без — оба варианта.
- Стены с присоединёнными другими стенами (joins) — L и T-узлы должны корректно определяться.

### M2. Выпуски (Dowels / Starter Bars)
**Цель.** Создать стержни-выпуски из нижележащей конструкции в стену-приёмник.

**Входы.**
- Выбор стен-приёмников.
- Автоматический поиск конструкций-источников по геометрии (плоскость, лежащая на низе стены) либо ручной выбор источника.
- Конфигурация выпусков (диаметр, шаг, длина анкеровки в источнике L_anc, длина в приёмнике L_lap, форма: прямой / Г-образный, защитный слой).

**Выходы.**
- Стержни `Rebar` с привязкой к стене-приёмнику (host = wall).
- Параметр `WRS_Bar_Role = "Dowel"` для последующей фильтрации.
- Связь со стеной-источником записывается в Extensible Storage: `DowelLinkSchema { SourceElementId, AnchorLengthIntoSource }`.

**Алгоритм определения источника.**
1. По нижней грани стены-приёмника строится горизонтальная плоскость с допуском ±tolerance.
2. Поиск элементов категорий `OST_StructuralFoundation`, `OST_Walls`, `OST_Floors`, `OST_StructuralFraming` (балки-ростверки) в окрестности плоскости.
3. Если найдено несколько кандидатов — открыть UI-диалог выбора.
4. Если не найдено — предупредить пользователя, предложить выбор вручную.

**Граничные случаи.**
- Источник тоньше требуемой анкеровки → автоматически переключиться на Г-форму с загибом.
- Несколько источников под одной стеной (стык фундамента и плиты) → создать отдельные выпуски для каждого участка.
- Источник имеет наклонную верхнюю грань → выпуски наклоняются перпендикулярно грани.

### M3. Назначение параметров (Parameter Assignment)
**Цель.** Записать в каждый стержень арматуры параметры, нужные для фильтрации на видах и группировки в спецификациях.

**Источники данных параметров.**
- Параметры стены (марка, ярус, секция, этаж, тип армирования).
- Параметры правила в конфигурации (роль стержня: вертикальный/горизонтальный/хомут/выпуск; позиция).
- Системные параметры арматуры (Type, Length, Volume).

**Перечень параметров (минимальный набор, полный — в `PARAMETERS.md`):**
- `WRS_Wall_Mark` — марка стены (копия из стены).
- `WRS_Wall_Tier` — ярус.
- `WRS_Wall_Section` — секция.
- `WRS_Wall_Level` — этаж (Name уровня).
- `WRS_Bar_Role` — роль (`Vertical`, `Horizontal`, `Edge`, `Diagonal`, `Stirrup`, `Dowel`, `Corner`, `Opening`).
- `WRS_Bar_Position` — позиция в маркировке (например, `1`, `2`, `K1`).
- `WRS_Config_Id` — идентификатор конфигурации.
- `WRS_Rule_Id` — идентификатор правила в конфигурации.
- `WRS_Run_Stamp` — отметка времени запуска (для отладки).

**Источник определения параметров.**
- Маппинг «параметр стены → параметр арматуры» задаётся в конфигурации в секции `parameter_mapping` (см. `CONFIG_SCHEMA.md`).
- Для производных параметров (`WRS_Bar_Role`, `WRS_Position`) значения определяются правилом, создающим стержень.

**Граничные случаи.**
- Параметр на стене отсутствует → подставить пустую строку и записать предупреждение в лог.
- Параметр на арматуре не существует в проекте → автоматически загрузить из Shared Parameter File плагина.

### M4. Виды и детали (Views & Details)
**Цель.** Создать в проекте необходимые виды для отображения армирования стены.

**Типы создаваемых видов.**
- **Поперечное сечение** (Section, разрез по толщине стены) — на каждый сегмент стены или по правилам.
- **Продольная развёртка** (Elevation / Section вдоль плоскости стены) — фасадный вид внешней/внутренней грани с проекцией арматуры.
- **Узлы** (Detail Section) — увеличенные сечения в местах:
  - сопряжения с фундаментом (выпуски),
  - L- и T-узлы стен,
  - окантовка проёмов.
- **3D-вид с фильтром «только арматура данной стены»** — изометрия для проверок.

**Источник параметров видов.**
- Имя стены (`Mark`) → имя вида: `АР-{Mark}_{Type}` (настраивается в конфиге).
- Шаблон вида — берётся из проекта по имени, задано в конфиге (`view_template`).
- Уровень детализации, фильтры — из шаблона.
- Масштаб — из конфига.

**Алгоритм.**
1. Для каждой выбранной стены вычислить bounding box и оси.
2. Создать `ViewSection` через `ViewSection.CreateSection(doc, viewFamilyTypeId, sectionBox)`.
3. Применить шаблон вида.
4. Назначить параметры вида (имя, на лист, группа).
5. (Опционально) автоматически разместить на листе по правилу из конфига.

**Граничные случаи.**
- Стена изогнутая → развёртка как «развёрнутый» вид (Unrolled): MVP — линейная проекция, v1.0 — по дуге.
- Стена с несколькими сегментами после сопряжений → решить по правилу: один вид на стену vs. один вид на сегмент.

### M5. Спецификации (Schedules)
**Цель.** Создать спецификации арматуры с группировкой по марке стены и другим параметрам.

**Типы спецификаций.**
- **Ведомость деталей арматуры** (Rebar Schedule) — позиция, диаметр, длина, форма, количество, общая длина, масса. Группировка по `WRS_Wall_Mark`.
- **Ведомость расхода стали** (ВРС) — суммарно по диаметрам, итого масса.
- **Спецификация выпусков** — отдельная для `WRS_Bar_Role = Dowel`.
- **Сводная ведомость по этажам / ярусам** — pivot по `WRS_Wall_Level` × `WRS_Bar_Diameter`.

**Источник конфигурации спецификаций.**
- Шаблоны спецификаций — в `templates/schedules/*.json` (см. `CONFIG_SCHEMA.md`).
- Каждый шаблон описывает: имя, категория, поля, фильтры, сортировку, форматирование, формулы.

**Алгоритм.**
1. Прочитать шаблоны спецификаций для текущей конфигурации.
2. Для каждого шаблона проверить, существует ли спецификация с таким именем в проекте.
3. Если нет — создать через `ViewSchedule.CreateSchedule()`.
4. Применить поля, фильтры, сортировку.
5. (Опционально) разместить на листах.

---

### M6. Конфигуратор (Config Editor)
**Цель.** UI для создания, сохранения, редактирования и применения конфигураций армирования.

**Сценарии.**
- Создать конфигурацию с нуля или скопировать существующую.
- Открыть существующую (из файла или из встроенной библиотеки).
- Сохранить как файл `.wrsconfig.json` локально или в централизованное хранилище (папка плагина).
- Импорт / экспорт между проектами.
- Редактирование секций: общие настройки, мешевая арматура, периметр, проёмы, углы, T-узлы, доп. рёбра, выпуски, маппинг параметров, шаблоны видов и спецификаций.
- Превью правил (схематическое 2D, неблокирующее).
- Валидация: предупреждения о коллизиях (например, шаг < диаметра × 2).
- Версионирование конфига (поле `version` и история изменений в комментариях).

**Связь со стеной.**
- Привязка конфига к стенам: команда **Wall Link** (выбор стен → выбор конфига).
- Хранение Wall Link в Extensible Storage стены.

## 4. Нефункциональные требования

| Атрибут | Требование |
|---|---|
| **Производительность** | Армирование 1 стены до 1 с; 50 стен — до 30 с. |
| **Транзакционность** | Все операции — в одной `Transaction`. Undo возвращает проект в исходное состояние. |
| **Совместимость** | Revit 2025, .NET 8. Опционально 2024 — через `#if REVIT2024`. |
| **Локализация** | UI ru-RU (основной), en-US (вторичный). Строки — в ресурсных файлах. |
| **Логирование** | Serilog, ротация по дням, путь `%APPDATA%\WRSPlugin\logs`. |
| **Стабильность** | Все исключения от Revit API обрабатываются, пользователю показывается `TaskDialog` с понятным сообщением. |
| **Тестируемость** | Сервисы покрыты unit-тестами (моки `Document` / `Wall`). Покрытие ≥ 60% MVP. |

## 5. Что **не входит** в scope первой версии

- Армирование колонн и балок — отдельный модуль, отложен.
- Сборные стеновые панели (precast) с раскладкой — позже.
- Автоматическое размещение видов на листах с генерацией штампов — отдельная фича.
- Импорт конфигов BeSmart — нет, формат проприетарный.
- Облачное хранилище конфигов — нет, только локальное / сетевая папка.
- Многопользовательская синхронизация конфигов — нет.

## 6. MVP-приоритеты

> MVP считается готовым, когда выполнены пункты с пометкой **[MVP]** ниже.

### MVP scope:
- [MVP] M1: External / Internal Reinforcement (основные сетки) для прямых вертикальных стен.
- [MVP] M1: Perimeter Reinforcement (краевые продольные).
- [MVP] M1: Opening Reinforcement — только Edge Rebar (без диагональных и хомутов).
- [MVP] M2: Прямые выпуски из фундаментной плиты (один источник).
- [MVP] M3: Назначение базовых параметров (`WRS_Wall_Mark`, `WRS_Bar_Role`, `WRS_Config_Id`, `WRS_Rule_Id`).
- [MVP] M4: Поперечное сечение + продольная развёртка по стене.
- [MVP] M5: Одна спецификация — Rebar Schedule с группировкой по `WRS_Wall_Mark`.
- [MVP] M6: Минимальный редактор конфига (форма + сохранение в файл) без превью.

### v1.0 scope:
- L-Corner и T-Connection Reinforcement.
- Opening Reinforcement: диагональные + U/O-хомуты.
- Additional Edge Rebar и Additional Reinforcement.
- Г-образные выпуски, несколько источников.
- Узлы (Detail Section) и 3D-проверочные виды.
- Полный набор спецификаций (ВРС, выпуски, pivot).
- Превью правил в редакторе конфига.
- Partial Update.

### Roadmap далее:
- Изогнутые стены, наклонные стены.
- Fabric Sheets.
- Импорт/экспорт конфигов в формат BeSmart-совместимый.
- Облачное хранилище конфигов.

См. подробности в `ROADMAP.md`.

## 7. Допущения и риски

| Допущение | Риск | Митигация |
|---|---|---|
| Revit 2025 API стабилен | Регрессии в Update | Привязка к конкретному билду API |
| Стены — категория `OST_Walls`, без `OST_Walls_Curtain` | Curtain-walls попадут в выборку | Фильтрация по `Wall.WallType.Kind == Basic` |
| Параметры на стене у пользователя могут отличаться от ожидаемых | Маппинг не сработает | Маппинг настраивается в конфиге, fallback — пустые значения + warning |
| Пользователь не использует Shared Parameters | Параметры арматуры нельзя создать | Плагин поставляется со своим Shared Parameter File и автоматически добавляет нужные параметры в проект при первом запуске |
