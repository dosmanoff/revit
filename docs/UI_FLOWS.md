# UX-сценарии и UI-описание

> Сопровождает: [SPEC.md](SPEC.md), [MODULES.md](MODULES.md).
> Документ описывает структуру Ribbon, основные диалоги и потоки пользователя.

## 1. Структура Ribbon

Tab: **WRS — Wall Reinforcement Suite**

```
┌─ Configurations ──────┬─ Walls ────────────────┬─ Dowels ──────┬─ Output ────────────────┐
│  [Config Editor]      │  [Link Wall]           │  [Create      │  [Create Views]         │
│  [Open Config Folder] │  [Arm Walls]           │   Dowels]     │  [Create Schedules]     │
│                       │  [Update Rebar]        │  [Update      │                         │
│                       │  [Modify Rebar]        │   Dowels]     │                         │
│                       │  [Delete Rebar]        │               │                         │
└───────────────────────┴────────────────────────┴───────────────┴─────────────────────────┘
```

Каждая кнопка — `PushButton` с иконкой 32×32 и tooltip с описанием и пиктограммой результата.

## 2. Команды и UI-описание

### 2.1 Link Wall
**Цель.** Привязать одну или несколько стен к конфигурации.

**Поток:**
1. Пользователь нажимает **Link Wall**.
2. Plugin переключает режим в `Selection.PickObjects(ObjectType.Element)` с фильтром по `OST_Walls`.
3. Открывается диалог `LinkWallView`:
   - **Слева:** список выбранных стен (Mark, Type, Length, Height) с чекбоксами.
   - **Справа:** список доступных конфигов (читается из `%APPDATA%\WRSPlugin\configs\` + заводские). Поиск по имени.
   - **Внизу:** кнопки `Apply`, `Cancel`, `Edit Config…` (открывает редактор для выбранной).
4. По `Apply`: на каждой стене записывается `WRS.WallLink`, параметр стены `WRS_Config_Id` обновляется.

**Edge cases:**
- Стена уже привязана к другому конфигу → строка подсвечивается жёлтым, требуется подтверждение перезаписи.
- Конфиг не подходит по `applicability.wall_kinds` или `min_thickness` → строка серая, иконка предупреждения.

---

### 2.2 Arm Walls
**Цель.** Сгенерировать арматуру в выбранных стенах.

**Поток:**
1. Если ничего не выбрано — `Selection.PickObjects` (категория Walls).
2. Открывается диалог `ArmWallView`:
   - **Tab 1: Walls** — список выбранных стен с состоянием:
     - `Mark`, `Type`, `Linked Config`, `Last Run`, `Status` (Idle / Has Rebar / Stale).
     - Чекбокс «выбрать все».
   - **Tab 2: Options:**
     - Use Wall Link / Pick config manually (radio).
     - При manual — выпадающий список конфигов.
     - Чекбоксы по разделам конфига: «External mesh», «Internal mesh», «Perimeter», «Opening», «L-Corner», «T-Connection», ... (Partial Update в v1.0).
     - «Перезаписать существующую арматуру плагина» (yes/no).
     - «Создавать выпуски» (yes/no) — отдельный модуль M2, можно запустить в той же транзакции.
   - **Tab 3: Preview** — список правил, которые сработают, с предполагаемым количеством стержней.
3. По `Apply`:
   - Прогрессбар (если стен > 5).
   - Запуск External Event → `WallReinforcementOrchestrator.Execute`.
   - По окончании — модальный `Report Dialog` со счётчиками (`Created`, `Updated`, `Deleted`, `Skipped`, `Warnings`, `Errors`) и кнопкой «Подробнее» (открывает текстовый лог).

**Edge cases:**
- Стена не имеет Wall Link → варнинг, предложение задать конфиг руками.
- Несовместимость конфига и стены → пропуск стены с warning.

---

### 2.3 Update Rebar
**Цель.** Пересоздать арматуру в стене по текущему привязанному конфигу.

**Поток:**
1. Pick walls.
2. Без диалога (или с короткой подтверждалкой) — запуск Orchestrator в режиме `Update`.
3. Внутри: diff между текущей арматурой и тем, что должно быть; обновление только различий.

**Партиальный Update (v1.0):** диалог выбора правил, которые надо пересоздать.

---

### 2.4 Modify Rebar
**Цель.** Точечная корректировка отдельных стержней с сохранением «связи» с правилом.

- Открывает Revit-стандартный режим редактирования (`PostCommand` на стандартную команду Revit) с уведомлением: «Изменения будут сохранены, при следующем Update этот стержень будет помечен как `WRS_Modified = Yes` и не пересоздан».

---

### 2.5 Delete Rebar
**Цель.** Удалить всю арматуру плагина в выбранных стенах.

**Поток:**
1. Pick walls.
2. Модальное окно подтверждения с счётчиком стержней.
3. Удаление по фильтру `WRS_Config_Id != null AND HostWallId in selection`.

---

### 2.6 Create Dowels
**Цель.** Создать выпуски.

**Поток:**
1. Pick walls (приёмники).
2. Диалог `DowelView`:
   - **Tab 1: Walls** — таблица: Wall Mark, Linked Config, Source candidates (число).
   - **Tab 2: Source detection:**
     - Auto-detect (default) / Manual.
     - Если Manual — для каждой стены кнопка «Pick source».
   - **Tab 3: Options:**
     - Override pattern (yes/no): если yes — поля анкеровки/перехлёста/формы редактируются прямо в диалоге, иначе из конфига.
     - Создавать только выпуски без сеток (если стена ещё не армирована).
3. По `Apply` — Orchestrator + транзакция, отчёт.

**Edge cases:**
- Не найден источник → требуется ручной выбор или skip.
- Несколько источников под одной стеной → таблица с разбиением «зоны → источник».

---

### 2.7 Config Editor
**Главное окно плагина.** Размер по умолчанию 1280×800, ресайз разрешён, минимум 1024×600.

**Layout:**
```
┌─────────────────────────────────────────────────────────────────────────────┐
│  [New] [Open] [Save] [Save As] [Import] [Export] [Validate]                 │
├──────────────────┬──────────────────────────────────────────────────────────┤
│  Sections        │  Header: Config Name | Author | Created/Updated          │
│  ─────────────   │  ────────────────────────────────────────────────────    │
│  ▸ General       │                                                          │
│    Mesh          │   < область редактирования секции >                      │
│    Perimeter     │                                                          │
│    Opening       │                                                          │
│    L-Corner      │                                                          │
│    T-Connection  │                                                          │
│    Add. Edge     │                                                          │
│    Add. Face     │                                                          │
│    Dowels        │                                                          │
│    Parameters    │                                                          │
│    Views         │                                                          │
│    Schedules     │                                                          │
│    Rebar Types   │                                                          │
│    Hooks         │                                                          │
├──────────────────┴──────────────────────────────────────────────────────────┤
│  Status bar: Validation: 0 errors, 2 warnings   |   schema_version: 1       │
└─────────────────────────────────────────────────────────────────────────────┘
```

**Секции** соответствуют разделам конфига (см. CONFIG_SCHEMA.md). Каждая секция — отдельный `UserControl`.

**Особенности UI секций:**
- **Mesh (External / Internal):** две вкладки рядом; превью схемы сечения справа (схематичный SVG, обновляется при изменении полей).
- **Perimeter:** табличный редактор (rows = строки правил Edge Rebar), плюс отдельные tabs для Diagonal и U/O Stirrups.
- **Opening:** табличный редактор Edge Rebar с фильтром по min/max width; tabs Diagonal и U/O Stirrups.
- **L-Corner / T-Connection:** табы Outer / Inner, табличный редактор bars.
- **Add. Edge / Add. Face:** табличные редакторы.
- **Dowels:** одна форма + список edge_overrides.
- **Parameters:** три таблицы: from_wall, from_rule, constants. Подсказка по подстановкам.
- **Views:** аккордеон по типам видов; для каждого — превью имени по pattern.
- **Schedules:** список + редактор полей/фильтров/сортировки (drag&drop порядок полей).
- **Rebar Types / Hooks:** простые таблицы.

**Валидация:**
- Поля с ошибками подсвечены красным, всплывающие подсказки с сообщением.
- Статусбар: количество ошибок/предупреждений; клик — переход к первой ошибке.

**Save / Save As:**
- Сохранение в `%APPDATA%\WRSPlugin\configs\<name>.wrsconfig.json` (по умолчанию).
- Перед сохранением — бэкап старого файла `.bak`.
- Изменение `version` и `updated_at`.

**Open:**
- Файл-пикер с фильтром `*.wrsconfig.json`.
- Доступ к заводским через отдельный диалог «Open from Library».

**Validate:**
- Кнопка в тулбаре. Запускает `ConfigValidator` — статический анализ без записи. Открывает диалог со списком ошибок/предупреждений с кликабельными ссылками на поля.

---

### 2.8 Create Views
**Цель.** Создать виды по выбранным стенам.

**Поток:**
1. Pick walls (или взять все стены с `WRS_Config_Id`).
2. Диалог `CreateViewsView`:
   - Чекбоксы по типам видов (Section, Elevation, Detail Nodes, 3D Check).
   - Перекрытие существующих видов: Skip / Overwrite / Rename.
   - Помещать на лист: yes/no + выбор шаблона листа.
3. Apply → отчёт.

---

### 2.9 Create Schedules
**Цель.** Создать или обновить спецификации.

**Поток:**
1. Без выбора стен (работает на уровне проекта).
2. Диалог `CreateSchedulesView`:
   - Список шаблонов спецификаций из текущего конфига (или выбранного).
   - Чекбоксы — какие создавать/обновлять.
   - Размещать на листе: опционально.
3. Apply → отчёт.

## 3. Состояния и индикаторы стены

В каждом диалоге, где есть список стен, у каждой строки — иконка состояния:

| Иконка | Состояние | Значение |
|---|---|---|
| ⚪ | Idle | Нет привязки к конфигу |
| 🔗 | Linked | Привязан конфиг, армирование не запускалось |
| ✅ | Up-to-date | Армирование совпадает с конфигом |
| ⚠ | Stale | Конфиг изменён после последнего запуска (по `RuleHash`) |
| ❌ | Error | При последнем запуске были ошибки |

## 4. Цветовая схема и стиль

- Используется встроенная Revit-светлая тема для совместимости (нет тёмной темы в Revit 2025 by default).
- Палитра:
  - Acccent: `#2D7DD2` (синий, кнопки Apply).
  - Warning: `#F39C12`.
  - Error: `#C0392B`.
  - Success: `#27AE60`.
  - Background panel: `#F4F4F4`.
- Шрифт: `Segoe UI`, размеры 12 / 13 / 14 pt.
- Все строки UI — через ResX (`Resources/Strings.resx`, `Strings.en-US.resx`).

---

## 5. Доступность и горячие клавиши

- Все основные команды доступны через Ribbon, плюс настраиваемые шорткаты (`KS` для Revit Keyboard Shortcuts).
- В диалогах — навигация по Tab, активация по Enter, отмена по Esc.
- Меньшая высота строк для таблиц = 24 px; колонки — ресайз и сортировка.

---

## 6. Локализация

- По умолчанию — ru-RU. Переключение через настройку плагина (`%APPDATA%\WRSPlugin\settings.json`).
- Все диалоги получают строки из ResX по локали Revit (`UIApplication.GetUILanguage()`), но можно переопределить.
